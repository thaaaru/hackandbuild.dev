1. Networking Fundamentals

TCP/IP Model

Application: HTTP, DNS, SMTP
Transport: TCP, UDP, Ports
Internet: IP Addressing, Routing
Link: MAC, ARP, Physical

IP Addressing & CIDR

  • IPv4 Format: 192.168.1.1
  • CIDR: /24 = 255.255.255.0
  • Private Ranges: 10.0/8, 172.16/12, 192.168/16

Common Ports

22 SSH, 53 DNS, 80 HTTP, 443 HTTPS, 3389 RDP

Tools & Commands

nmap -sC -sV <target>

tcpdump -i eth0 port 80

ip addr (Linux), ipconfig /all (Win)

2. Operating Systems

Linux Core Skills

  • Navigation: ls, cd, find, cat
  • Permissions: chmod 4755 file (SUID)
  • Services: systemctl status sshd
  • Networking: ss -tuln, ip route
  • Logs: /var/log/auth.log

Windows Core Skills

  • PowerShell: Get-Service, Get-Process
  • Event Logs: Get-WinEvent -LogName Security
  • User Mgmt: net user, whoami /priv
  • Services: sc query

Practical Tasks

Linux: Find SUID binary for privesc.

Windows: Find cleartext creds in scheduled tasks.

3. Web Technologies

HTTP Methods & Status

Methods: GET, POST, PUT, DELETE

Codes: 200 (OK), 302 (Redirect), 403 (Forbidden), 404 (Not Found), 500 (Server Error)

Cookies & Sessions

  • Attributes: HttpOnly, Secure, SameSite
  • Risks: Session fixation, XSS stealing tokens.

Data Flow Threat Points

User Input ➔ Client ➔ HTTP Request ➔ Server ➔ DB ➔ Response

Tools

Browser DevTools (Inspect, Network)

Burp Suite (Intercept, Replay, Fuzz)

4. Programming & Scripting

Why Scripting Matters

Automate scans, parse logs, craft custom payloads, and chain tools together.

Python Basics

  • requests for HTTP
  • re (regex) for parsing
  • socket for custom clients

Bash & PowerShell

Bash: Use awk, sed, grep to process text.

PowerShell: Object-based pipeline, Invoke-RestMethod.

Best Practices

Use rate limits, avoid hardcoding credentials, and always scan politely.

5. Security Concepts

Auth & Access Control

Authentication: Who you are.

Authorization: What you can do.

Encryption

  • Symmetric vs. Asymmetric
  • TLS: Secures data in transit.

OWASP Top 10 (Concepts)

  • A1: Injection (e.g., SQLi)
  • A2: Broken Authentication
  • A5: Broken Access Control
  • A7: Cross-Site Scripting (XSS)

6. Legal & Ethics

Core Ethical Rules

  • Get Written Consent: Never test without permission.
  • Define Scope: Know what's in and out of bounds.
  • Respect Privacy: Avoid accessing or exfiltrating real user data.

Rules of Engagement

Document allowed methods, testing hours, and emergency contacts before starting.

Responsible Disclosure

Report findings privately to the owner, provide clear PoCs, and agree on a timeline for fixing.
